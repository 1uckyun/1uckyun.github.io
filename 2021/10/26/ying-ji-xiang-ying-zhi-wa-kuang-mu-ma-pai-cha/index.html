二、排查流程-Linux

1、日志检查
# 错误登录日志
last -f /var/log/btmp
# 成功登录日志
last -f /var/log/wtmp
# 历史命令执行情况
cat ~/.bash_history
# 记录系统中所有用户最后一次登录时间的日志
lastlog

2、ssh检查
# 查看是否存在异常的ssh公钥
ls -al ~/.ssh/

3、进程检查
# 查看资源占用情况
top -c
# 查看指定进程问津路径
ls -l /proc/$PID/exe
# 杀死恶意进程
kill -9 $PID

4、文件检查
# 查找5天内修改过的所有文件
find / -mtime 5 -name '*'
# 查找5天内新增的所有文件
find / -ctime 5 -name '*'


5、防火墙检查
# 查看是否存在异常的防火墙规则
iptables -L -n
# 阻断挖矿木马网络同学
iptables -A INPUT -s 可疑地址 -j DROP 
iptables -A OUTPUT -d 可疑地址 -j DROP

6、连接检查
# 查看当前连接情况
netstat -antp

7、计划任务检查
# 查看指定用户的计划任务
crontab -u username -l
# 其他计划任务文件位置
cat /etc/crontab 
cat /var/spool/cron 
cat /etc/anacrontab 
ls -al /etc/cron.d/ 
ls -al /etc/cron.daily/ 
ls -al /etc/cron.hourly/ 
ls -al /etc/cron.weekly/ 
ls -al /etc/cron.monthly/ 
ls -al /var/spool/cron/

8、系统启动项检查
# centos7以下版本
chkconfig –list
# centos7以上版本
systemctl list-unit-files
# centos7以下版本关闭恶意启动项
chkconfig 服务名 off
# centos7以上版本关闭恶意启动项
systemctl disable 服务名
# 其他系统启动项目录（可按修改日期先后顺序查看）
ls -al /usr/lib/systemd/system 
ls -al /usr/lib/systemd/system/multi-user.target.wants 
ls -al /etc/rc.local 
ls -al /etc/inittab 
ls -al /etc/rc0.d/ 
ls -al /etc/rc1.d/ 
ls -al /etc/rc2.d/ 
ls -al /etc/rc3.d/ 
ls -al /etc/rc4.d/ 
ls -al /etc/rc5.d/ 
ls -al /etc/rc6.d/

三、排查流程-Windows

1、异常用户排查
lusrmgr.msc
HKEY_LOCAL_MACHINE\SAM\SAM\Domains\account\Users\Names

2、事件日志排查
eventvwr.msc

3、计划任务排查
HKEY_CURRENT_USER\software\micorsoft\windows\currentversion\run 
HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Run 
HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\RunOnce
gpedit.msc -> windows设置 -> 脚本

4、异常端口排查
netstat -ano 
tasklist | findstr PID
